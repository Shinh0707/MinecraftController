Most blocks have a corresponding item form, which is the item most closely associated with the block, used for placing said block into the world, and represents it in perhaps more obscure cases, such as the Superflat customization screen, Enderman drops and the ability to have the block state which is placed customized. Special cases and exceptions to this rule have colored backgrounds, detailed in the below table.


Default This block has an item form with an identical ID.
Blue This block has an item form with a differing ID.
Purple This block is an alternate placement of a related block, and has an item form with an identical ID with the related block.
Yellow This block does not have a direct item form, but an indirect item form can be obtained with the Pick Block function.
Red This block does not have an item form at all.
